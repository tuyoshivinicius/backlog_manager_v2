## Problema Identificado

**Anomalia**: CRITICAL - Baixa taxa de persistencia
**Metrica**: stories_allocated (banco) = 90 vs stories_allocated (algoritmo) = 189
**Threshold**: banco deve refletir resultado do algoritmo
**Descricao**: O script `extract_metrics.py` executava a alocacao e reportava metricas, mas nao persistia as stories alocadas no banco de dados. O algoritmo alocava 189/190 stories corretamente, mas ao verificar o banco, apenas 90 estavam alocadas (historico de execucoes anteriores).

## Diagnostico

**Causa Raiz**: O script `extract_metrics.py` foi projetado apenas para extrair metricas de analise, executando a alocacao em memoria sem salvar os resultados. Diferente do `ExecuteAllocationUseCase` que persiste via `uow.stories.update()`, o script terminava sem commit das mudancas.

**Arquivos Envolvidos**:
- `scripts/extract_metrics.py`
- `src/backlog_manager/application/use_cases/allocation/execute_allocation.py` (referencia)

## Mudanca Aplicada

**Arquivos Modificados**:
- `scripts/extract_metrics.py`: Adicionada flag `--persist` que opcionalmente salva as stories alocadas no banco

**Codigo Alterado**:
