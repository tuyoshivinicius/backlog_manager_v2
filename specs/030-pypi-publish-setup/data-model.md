# Data Model: PyPI Publish Setup

**Feature Branch**: `030-pypi-publish-setup`
**Date**: 2026-03-31

## Overview

This feature has no domain entities, database changes, or application-layer models. It operates exclusively at the packaging/distribution layer. The "entities" below describe the configuration and artifact structures involved.

## Entity: Package Metadata (pyproject.toml)

**Location**: `pyproject.toml` — `[tool.poetry]` section

| Field | Type | Value | Constraint |
|-------|------|-------|-----------|
| name | string | `"zion-backlog-manager"` | PyPI distribution name; must be unique on PyPI |
| version | string | `"0.1.0"` | Semver; single source of truth |
| description | string | English description | FR-007B |
| authors | list[string] | `["Tuyoshi Vinicius <tuyoshi.dev@gmail.com>"]` | FR-003 |
| license | string | `"MIT"` | FR-004 |
| readme | string | `"README.md"` | Existing |
| homepage | string | GitHub URL | FR-005 |
| repository | string | GitHub URL | FR-005 |
| keywords | list[string] | Domain keywords | FR-007 |
| classifiers | list[string] | PyPI classifiers | FR-006 |
| packages | list[object] | `[{include = "backlog_manager", from = "src"}]` | Unchanged — internal namespace preserved |

## Entity: CLI Entry Point

**Location**: `pyproject.toml` — `[tool.poetry.scripts]`

| Field | Value | Notes |
|-------|-------|-------|
| `zion-backlog-manager` | `backlog_manager.presentation.app:main` | New entry; replaces `backlog-manager` |

## Entity: Build Artifacts

**Generated by**: `poetry build`

| Artifact | Naming Pattern | Example |
|----------|---------------|---------|
| sdist | `zion_backlog_manager-{version}.tar.gz` | `zion_backlog_manager-0.1.0.tar.gz` |
| wheel | `zion_backlog_manager-{version}-py3-none-any.whl` | `zion_backlog_manager-0.1.0-py3-none-any.whl` |

**Validation rule**: Publish script MUST verify `zion_backlog_manager` appears in artifact filenames before upload.

## Entity: Publish Script Configuration

**Location**: Environment variables (runtime)

| Variable | Purpose | Required |
|----------|---------|----------|
| `PYPI_TOKEN` | PyPI API token (preferred) | One of PYPI_TOKEN or TWINE_USERNAME+TWINE_PASSWORD |
| `TWINE_USERNAME` | PyPI username or `__token__` | Alternative to PYPI_TOKEN |
| `TWINE_PASSWORD` | PyPI password or API token | Paired with TWINE_USERNAME |

## State Transitions

Not applicable — this feature has no stateful entities.

## Relationships

```
pyproject.toml metadata
    └── poetry build → Build Artifacts (dist/)
        └── twine upload → PyPI Registry
```
